# NeetResearch App

> A full-stack AI research platform that orchestrates multi-agent workflows to generate structured, source-backed reports with real-time progress tracking.

## What Is This?

NeetResearch is an agent-based research system for running deep, structured research tasks from a simple UI. You enter a topic, the backend routes work through specialized agents and tools, and the app returns a final report with citations and export options.

This project is designed as a practical system, not a single-prompt demo. It includes orchestration, tool use, session tracking, and real-time frontend updates.

---

## Features

### 🤖 Multi-Agent Research Orchestration
- Graph-based research flow with scoping, supervision, research, and report generation stages
- Specialized agents for planning, delegation, and evidence synthesis
- Gemini-backed model calls with resilient handling for retries and fallbacks

### 🌐 Web + Database Research Modes
- Web research using ADK/web tools for source gathering and summarization
- Database retrieval mode using local document search
- Hybrid mode to combine both pipelines

### 📡 Real-Time Progress Tracking
- Live session updates through WebSocket endpoints
- HTTP polling fallback for resilience during connection drops
- User-visible phase progress (initializing, scoping, researching, analyzing, generating)

### 📄 Report Generation and Export
- Final report creation with source-backed narrative output
- Report view page with markdown rendering
- Export support for Markdown and PDF

### 🧠 Session Management and Caching
- Start, resume, cancel, and review research sessions
- Final report cache path for repeated queries
- Session status and history browsing in the frontend

### 🛡️ Professional Error Handling UX
- Friendly error summaries for users
- Expandable **More details** view for technical diagnostics
- Standardized frontend error normalization (including quota/rate-limit cases)

## Tech Stack

| Layer | Technology | Why |
|---|---|---|
| Frontend | React + TypeScript + Vite | Fast UI iteration and strong typing |
| State | Zustand | Lightweight state management for sessions and toasts |
| Routing | React Router v6 | SPA route orchestration |
| Backend API | FastAPI | Async-first Python API with strong performance |
| Agent Runtime | LangGraph + Google ADK | Structured multi-agent workflows and tool invocation |
| LLMs | Google Gemini models | High-quality reasoning and synthesis |
| Storage | ChromaDB + local cache | Retrieval support and repeated-query acceleration |
| Transport | REST + WebSockets | Reliable request/response and real-time updates |

## Architecture

![Architecture Diagram](https://dummyimage.com/1280x720/0f172a/cbd5e1.png&text=Architecture+Diagram)

### Request Flow

1. User submits a topic from frontend.
2. Backend creates a research session.
3. Research graph runs through scoping and supervision.
4. Agents call web/database tools and gather notes.
5. System generates final report and stores session state.
6. Frontend receives progress updates via WebSocket/HTTP and renders results.

## Known Demo Limitation

- During demo usage, Google Gemini free-tier limits can occasionally return `429 RESOURCE_EXHAUSTED`.
- The frontend handles this with a friendly message and an expandable **More details** panel for diagnostics.
- If it appears, wait for the cooldown shown in the message and retry, or use a higher-quota API key for uninterrupted demos.

## Getting Started

### Prerequisites
- Python 3.10+
- Node.js 18+
- Google Gemini API key (for agent model calls)

### 1. Clone and Setup

```bash
git clone https://github.com/yourusername/neetresearch-app.git
cd neetresearch-app
```

### 2. Backend Setup

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements-backend.txt
```

### 3. Frontend Setup

```bash
cd frontend
npm install
cd ..
```

### 4. Environment

Create a `.env` file in project root (or update existing one) with your keys/settings:

```env
GOOGLE_API_KEY=your_api_key_here
```

### 5. Run Locally

```bash
# Terminal 1 - Backend
.venv\Scripts\python -m uvicorn Scripts.App.api.main:app --reload --host 127.0.0.1 --port 8000

# Terminal 2 - Frontend
cd frontend
npm run dev
```

Frontend: `http://localhost:5173`  
Backend: `http://127.0.0.1:8000`
